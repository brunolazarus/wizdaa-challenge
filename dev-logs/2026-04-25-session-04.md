### Key Design Decisions Made

**`justUpdated` detection via version ref (not balance value):**
`BalanceTable` has an internal `BalanceRow` component. A `useRef` tracks the previous `data?.version`. A `useEffect` watching `data?.version` sets a `justUpdated` boolean for 3s when the version changes. Using version (not raw balance) is more correct — version is the HCM-authoritative change signal.

This `useEffect` tracks a *transient local UI flag*, not server data. It is not a violation of the "no useEffect to sync server data into state" rule — that rule targets copying server data values into `useState` to avoid React Query becoming the source of truth.

**`RequestForm` owns `useSubmitRequest`:**
`RequestStatusBanner` and `StaleDataWarning` are rendered from within `RequestForm` (not the page). The page just renders `<RequestForm>`. This keeps the mutation state and its visual effects co-located, avoids prop-drilling status, and makes `RequestForm` a self-contained feature.

**`BalanceRefreshedMidSession` story uses a stateful closure in MSW:**
A `callCount` counter in the handler closure returns the base balance on the first call and an incremented version on subsequent calls. This simulates a background HCM mutation landing between polls without needing to artificially shorten the polling interval.

**Storybook interaction tests for the two critical paths (from CLAUDE.md):**
- `OptimisticRolledBack` story: fill form → submit → MSW 422 → `findByText('Request rejected')`
- `HcmSilentlyWrong` story: submit → MSW 200 with same version → `findByText('Unconfirmed')`
