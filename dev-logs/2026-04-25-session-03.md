### Key Design Decisions Made

**`useMutationState` predicate for per-key guard (not `useIsMutating` with `mutationKey`):**
`useSubmitRequest` is a no-arg hook — the `[employeeId, locationId]` pair isn't known at definition time. Instead of a static `mutationKey`, `useBalance` uses `useMutationState` with a `predicate` that inspects `mutation.state.variables` cast to `HcmWriteRequest`. This gives per-key filtering without coupling the hooks via a shared key.

```ts
useMutationState({
  filters: {
    status: 'pending',
    predicate: (mutation) => {
      const vars = mutation.state.variables as HcmWriteRequest | undefined
      return vars?.employeeId === employeeId && vars?.locationId === locationId
    },
  },
})
```

**`optimistic-pending` is derived, not stored (TRD §2.5):**
`useSubmitRequest` stores only terminal states in `useState`. `optimistic-pending` is derived from `mutation.isPending`. This keeps the state in sync with React Query's source of truth.

```ts
const status: RequestStatus = mutation.isPending ? 'optimistic-pending' : terminalStatus
```

**`fetchQuery` for read-after-write verification:**
`onSuccess` calls `queryClient.fetchQuery({ staleTime: 0 })` to force a fresh GET from primary. `staleTime: 0` guarantees the queryFn always runs (never serves from cache). Version comparison after this fetch determines `hcm-confirmed` vs `hcm-silently-wrong`.

**Server Component hydration (no `useEffect`, no loading spinner):**
`app/layout.tsx` is now async. It calls `getBalances(getStore())` directly (same process, no HTTP round-trip), pre-populates both `queryKeys.balances()` and each `queryKeys.balance(empId, locId)` per-cell entry, then dehydrates into `HydrationBoundary`. First paint shows data.

**`await vi.advanceTimersByTimeAsync(0)` to flush async `onMutate`:**
`onMutate` is async and uses `await cancelQueries`. After calling `mutate()`, the optimistic update isn't synchronously visible — it's applied in a microtask. `await vi.advanceTimersByTimeAsync(0)` advances time by 0ms AND flushes all pending async tasks, making the optimistic update and `isMutating: true` state visible before the timer advance.
